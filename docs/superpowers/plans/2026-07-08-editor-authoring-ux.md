# Editor Authoring-UX Overhaul — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the seven authoring-UX improvements from
`docs/superpowers/specs/2026-07-08-editor-authoring-ux-overhaul-design.md` — a
bigger code box, a visual slide-template gallery, a schema-driven clickable
palette, `* +` bullet fragments + a frag-wrap action, a mode-aware menubar chrome
over a bottom-docked slide, and doc-settings edited through the main panel.

**Architecture:** Two DSL/renderer changes ship everywhere (`build.py`,
`grammar.py`); everything else is dev-editor-only JS/CSS under
`src/revealer/data/js/editor/` (served by `revealer serve`, excluded from built
decks by `assets.py`). The editor shares state via the `window.RV` bus; new UI
reuses `RV.ui.box`, `F.rvPostEdit`, `F.fetchSrc`, and `outline.js`/`format.js`
helpers rather than re-implementing them.

**Tech Stack:** Python 3 (renderer, grammar, pytest golden tests), vanilla ES5
editor modules, the in-browser JS test harness (`data/js/test/suite-*.js` driven
by `tests/test_editor_js.py` via headless Chrome), reveal.js runtime.

## Global Constraints

- **Editor-only scope:** new JS modules must early-return unless `window.__RV_DEV__`
  (see every existing `editor/*.js`) and must be registered in `assets.py`'s served
  bundle; they must NOT be injected into built decks (`inject_revealer_assets`).
- **i18n:** every user-facing string goes through `RV.t(...)` with an `en` and a
  `fr` entry in `editor/i18n.js`; escape with `RV.esc`/`F.escapeHtml`.
- **Grammar is the single source of truth:** DSL shapes/cheat metadata live in
  `grammar.py`; `build.py`/`edit.py`/the schema/the docs reference derive from it.
- **Docs must build `-W`-clean** (`myst_heading_anchors=3`); the reference chapter
  is generated by `Documentation/gen_reference.py`.
- **Edit ops** go through `F.rvPostEdit([...])` (sha/FIFO/toasts + SSE reload);
  never mutate `.pres` files directly from JS.
- **Preserve load-bearing hooks:** `#rv-ed-outline` id + `F.toggleOutline` toggle
  semantics (for `saveStateAndReload`/`restoreState` + `?rv-outline=1`), the
  `rv-ed-lastsave` save handshake, the git-`fallback` history dimming, keyboard
  shortcuts (`e`/`o`/`f`/Ctrl+Z), and reveal.js aspect-fit ownership of `.slides`.
- **Commit per task**; run the relevant tests before each commit; branch is
  `editor-authoring-ux`.

def _html(src):
    return build_pres(src)  # returns compiled slide HTML

def test_star_plus_makes_li_a_fragment():
    html = _html("=== T\n\n* + one\n* two\n")
    assert '<li class="fragment">one</li>' in html
    assert '<li>two</li>' in html            # unflagged bullet unchanged

def test_star_plus_index():
    html = _html("=== T\n\n* +2 one\n")
    assert 'class="fragment"' in html and 'data-fragment-index="2"' in html

def test_nested_and_dash_markers():
    html = _html("=== T\n\n* a\n  * + nested\n- + dash\n")
    assert html.count('class="fragment"') == 2

def test_plain_plus_bullet_is_not_a_fragment():
    html = _html("=== T\n\n+ ordinary plus bullet\n")
    assert 'fragment' not in html

def test_escaped_plus_is_literal():
    html = _html("=== T\n\n* \\+ literal\n")
    assert 'fragment' not in html and '+ literal' in html
```

# escaped "\+" already handled by the markdown escape layer before this point;
# ensure the marker regex runs on the escaped-stripped text so "\+" stays literal.
li_html = '<li{cls}{attr}>{body}</li>'.format(cls=li_cls, attr=li_attr, body=render_inline(item_text))
```
